Intro to HTML

  • HTML stands for hypertext markup language
  • HTML is the building block of the web
  • Organizes text, pictures on web
  • <!doctype html> : All web documents start with this. Tells browser to render HTML
  • <html> and </html> : Starts after doctype tag, ends at end of HTMl document. Everything in between is HTML code
  • <body> and </body> : Where most content goes. Not actually required, as page assumes other content is body
  • But good practice to do so.
  • <head> section usually comes before the <body>.
  • 2 different metaphors:
    • HTML is the skeleton / foundation
    • CSS is the skin / style
    • JS is the brain / functionality and behavior

Text Editors

  • Integrated Development Environments (IDE's) - more advanced text editors with more features

Text Elements I

  • A HTML element is formed using HTML tags
  • Basic structure consists of 4 key items
    1. Opening/Left Tag
    2. Content - b/w opening and closing tag
    3. Closing/Right tag
    4. (Optional) - Attribute name (e.g. id or class) and value
  • 2 Types of of HTML Elements:
    • Block Elements
      • <p> Paragraphs
      • <li> Lists
      • <h1> to <h6> Headings
      • <article> Articles
      • <section> Sections
      • <blockquote> Long quotes
      • <div> div Often used as a container for other HTML elements
      • Block elements structure main parts of page, dividing content into coherent blocks
      • Block-level elements always starts on a new line and takes up FULL width available
    • Inline Elemennts
      • <a> Links
      • <em> Emphasized words
      • <strong> Important words
      • <span> Inline element used to markup a part of a text or part of document
      • Inline elements meant to affect text and give it fx or meaning. Usually single or few words.
      • Does not start on a new line and only take sup as muhc width as necessary

Line Breaks

  • Spacing between code in HTML file does not affect positioning of elements in browser
  • To accomplish line breaks, can use the <br> tag
  • <br> tag can be used within the opening and closing p tag

HTML Lists

Unordered Lists

  • Lists items in no particular order, with bullet points
  • Uses the <ul> and </ul> to open and close ul list
  • Uses the <li> and </li> tags for each list item
  • Here is an example or an unordered list:
    • dog
    • cat
    • racoon

Ordered Lists

  • Lists items in a specific order
  • Uses the <ol> and </ol> to open and close ol list
  • Uses the <li> and </li> tags for each list item
  • Here is an example of an ordered list
  1. Take out the recycling
  2. Go for a run!
  3. Pickup baby from daycare

Attributes

Images

  • src attribute - URL or file location
  • alt attribute - image's alternative text (description for those with accessibility needs)
  • width and height attributes - used to adjust image size
  • images are self-closing - add slash at end, instead of another </img>
  • Example: <img src="images/nefertiti.jpg" alt="A smiling labrador retriever" width="480" height="320 >

Links

  • Inline element written with a <a> tag. href attribute (hypertext reference) defines definition of link
  • 3 types of destinations you can define
    • anchor targets - navigate within same page
    • relative URLS - navigate within same website
    • absolute URLS - navigation to another website
  • Can also use additional attribute beside "a" and "href"
    • rel attribute - specifies rleationship b/w current and linked document
    • target attribute - specifies where to open linked document
    • target="_blank" - opens link in NEW window

Comments

  • Writes code without hcanging how website displays
  • Ignored by browser and only helpful to coders
  • Comment starts with <!-- and ends with -->

Self-Enclosing Elements

  • Some elements only have an opening tag
  • These include:
    • <br>
    • <img src="panda.png" alt="Description">
    • <input type="text">

Other Attributes

  • class="apple" can be styled using CSS .apple {color:red}
  • id="apple" can be styled using CSS #apple {color:red}

The DOM

Intro to the DOM:

  • DOM = Document Object Model
  • Model that the web document is presented as
  • Like a tree with branches that end with a node
  • Each node contains objects
  • Holds all the elements in the order that we place them
  • Starts with parent items that branch into child items
  • Starts with HTML (top node - level 1), then head and body (level 2), etc.. (

Setting up an HTML file

  • <!DOCTYPE html>
    • First line of HTML code, tells browser to expect HTML
    • Refers to HTML5, as the current standard
  • <html>
  • The Head
    • <head> - The head element goes above the body tag
  • Metadata
    • <metadata> -The head element contains the metadata tag
    • Displays info about page not displayed directly on web page
    • Contains metadata of the webpage
  • Title
    • <title> - Title tag always inside the head tag
    • This title is dipplayed in the brower's title bar (or in the tab's title)
  • Body
    • Contains various types of content including - text, images, buttons

Hierachy

  • When element is contained inside another element, it is considered the "child" of the element
  • Child elements are nested inside of the parent element
  • Multiple levels of nesting, means analogy can be extended to grandchildren, great-grandchildren, etc...
  • Relationships b/w ancestor and descendent elements = hierarchy
  • Child elements inherit behavior and styling from their parent elements

Semantic Elements

  • Structure elements used to organize main parts of page. Usually contain other HTML elements
  • Typical website includes:
    • header - 1st leement, includes logo and tagline
    • nav - list of links that go to different parts of website
    • h1 - title of the page
    • article - main content of the page, like a blog post
    • footer - last element of page, located at bottom
